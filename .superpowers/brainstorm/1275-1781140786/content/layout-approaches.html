A — Auto-ocultar en rutas operativas

Al entrar a /farmacia o /caja, el sidebar se oculta automáticamente. El header muestra siempre un botón ☰ Menú que lo abre como overlay temporal. Al salir de esas rutas, el sidebar vuelve a aparecer normalmente.

Pros

  • Sin acción del usuario
  • Máximo espacio POS
  • Consistente

Contras

  • Puede sorprender la primera vez

B — Sidebar colapsado (solo íconos)

El sidebar no desaparece — colapsa a una tira de 40px con solo íconos. El usuario puede expandirlo con un clic. Estado recordado por ruta en localStorage.

Pros

  • Navegación siempre visible
  • Ya existe isCollapsed

Contras

  • Pierde ~40px de espacio
  • Menos dramático

C — Botón flotante (FAB)

Pantalla completamente libre, con un botón circular flotante en la esquina inferior izquierda. Al pulsarlo se abre el sidebar como overlay. Minimalista pero puede chocar con contenido del POS.

Pros

  • 100% pantalla libre
  • Familiar (móvil)

Contras

  • Puede tapar contenido
  • Inconsistente con el estilo actual
