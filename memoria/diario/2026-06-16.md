## Aprendizaje permanente
- Antes de diseñar tabla nueva, revisar si ya existe algo deployado en prod sin documentar — `entregas_turno` y `solicitudes_insumos` ya existían y ahorraron una migración completa
- Para nurse_categoria enum + función con mismo nombre: `CREATE TYPE` + `CREATE FUNCTION` fallan si se hacen en transacciones separadas tras un primer intento fallido (rollback total) — aplicar todo en un solo `apply_migration`
- Patrón de prueba en vivo sin browser: simular JWT con `set_config('request.jwt.claims', json_build_object('sub', user_id)::text, true)` + `set_config('role','authenticated', true)` antes de invocar RPC con `has_role()`/`auth.uid()`
- Resultado de MCP que excede límite de tokens se autoguarda a archivo — unwrap `{"result":"..."}` + regex `/\[\{.*\}\]/s` + node para extraer el JSON real
- `node -e` con `require()` falla si `package.json` tiene `"type":"module"` — usar extensión `.cjs`
- SQL Server: columna usada como PK no puede ser `NVARCHAR(MAX)` — acotar a tamaño fijo (ej. `NVARCHAR(450)`)
- `sqlcmd` en modo texto rompe con contenido que incluya tokens como `GO` en líneas propias — usar ADO.NET parametrizado para volcar contenido arbitrario (código, texto libre)
