### 2. Fix pérdida de estado al cambiar ventana/ruta
Diagnóstico de 3 causas:
- **Causa A (crítica)**: Supabase puede disparar `SIGNED_IN` aunque el usuario ya estaba autenticado (al volver a tab, refresh de sesión). Esto ponía `loading=true` → `ProtectedRoute` mostraba spinner → toda la app se desmontaba → estado perdido
- **Causa B (media)**: navegación entre rutas desmonta componentes — React normal, pero doloroso en formularios
- **Causa C (baja)**: tab discard del navegador

**Fix 1 — useAuth.tsx**: `wasAuthenticatedRef` trackea si ya había sesión. Si `SIGNED_IN` llega mientras ya autenticado → `fetchRoles()` silencioso sin `setLoading(true)`. Cero desmonte de páginas.

**Fix 2 — NuevaCita.tsx**: form draft en `sessionStorage["nueva-cita-draft"]`. Se restaura al montar, se limpia al enviar o cancelar.

**Fix 3 — DetalleCita.tsx**: monto Stripe en `sessionStorage["stripe-amount-{citaId}"]`. Se restaura al montar, se limpia al completar pago.

### 3. Análisis Stripe Projects
Usuario preguntó sobre `docs.stripe.com/projects`. Conclusión: útil para proyectos nuevos desde cero, no vale la migración para este proyecto ya maduro.
